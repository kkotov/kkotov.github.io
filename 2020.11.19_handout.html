  • modifying mutable variables

Another paradigm:

Algebra (elements and operators transforming these elements to … elements) allows to reason about higher-level abstractions like polynomials, collections, shapes, strings …, e.g.:

$$ (a\cdot x + b) + (c\cdot x + d) = (a+c)\cdot x + (b+d) $$

$$ (string_1 + string_2) + string_3 = string_1 + (string_2 + string_3) $$

Functional programming:

  • without mutable variable
  • without assignments
  • without loops and other imperative control structures

Achieved with algebra of functions: functions take functions as arguments and return functions

Syntactic beauty of functions

Take sums between a and b, \(\sum_{n=a}^{b}f(n)\) and simplify it step by step:

def sumInts(a: Int, b: Int): Int = if (a > b) 0 else a + sumInts(a + 1, b)

def cube(x: Int): Int = x * x * x
def sumCubes(a: Int, b: Int): Int = if (a > b) 0 else cube(a) + sumCubes(a + 1, b)

def factorial(x: Int): Int = if (x == 0) 1 else x * factorial(x - 1)
def sumFactorials(a: Int, b: Int): Int = if (a > b) 0 else factorial(a) + sumFactorials(a + 1, b)

def sum(f: Int => Int, a: Int, b: Int): Int = if (a > b) 0 else f(a) + sum(f, a + 1, b)

def sumInts = sum(x => x, a, b)
def sumCubes = sum(x => x * x * x, a, b)

def sum(f: Int => Int): (Int, Int) => Int = { 
    def sumFunc(a: Int, b: Int): Int =
        if (a > b) 0
        else f(a) + sumFunc(a + 1, b)
    sumFunc
}
sum(x => x)(1,10) + sum(x => x * x * x)(1,10) + sum(factorial)(1,10)

Immutability of data, is it possible? (BST example)

*** =left

abstract class BST {
  def contains(x: Int): Boolean
  def include(x: Int): BST
}

class Empty extends BST {
  def contains(x: Int): Boolean = false
  def include(x: Int): BST = new NonEmpty(x, new Empty, new Empty)
}

class NonEmpty(value: Int, left: BST, right: BST) extends BST {

  def contains(x: Int): Boolean =
    if (x < value) left.contains(x)
    else if (x > value) right.contains(x)
    else true

  def include(x: Int): BST =
    if (x < value) new NonEmpty(value, left.include(x), right)
    else if (x > value) new NonEmpty(value, left, right.include(x))
    else this
}

Applications

  • Functional programming focuses on “instructions” to be performed on data rather than on data themselves. It makes it ideal instrument for horizontally scalable applications (e.g. MapReduce) where you distribute “instructions” over agents and let them work in parallel.

  • Scala is the workhorse of Spark

  • Many languages offer support for functional programming (while not being “pure” functional languages). For example, Python offers functional semantics and that also allows it to be used with Spark (PySpark project).

  • Some parts of Scala’s syntax happened to be adopted in other languages (e.g. Golang)

  • We already use C++ template metaprogramming in our code base


Want to learn more? Look at [Scala Specialization](https://www.coursera.org/learn/progfun1) from its creator in EPFL.
